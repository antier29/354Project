Modules, Higher Order Functions, Array Methods


Modules


Modules are used when you want to split up your files. This way your code is smaller and easier to use. It is also helpful for when you want to edit the specific code without having to edit your entire block of code. You can access the modules variables and functions by using IMPORT and EXPORT. Export: where variables functions labels and keywords are attainable from outside of the module. Import: you get access to the components of other modules Why are they important? Modules are simply a smaller portion of code that you can reuse. You can use modules as classes in an object-oriented programming language. In order for import/export to work you need to have. script type=“module” in your html. 

            //sayHi.js
            export function sayHi(user)
            {
	            alert (`Hello, ${user}!`);
            }

            //main.js
            import {sayHi} from ‘./sayHi.js’;

            alert(sayHi); //function
            sayHi(‘ash’); //Hello, ash!

Higher Order Functions

How do they work? The definition of a higher order function says that it accepts a function as an argument and it can return a function. If you have a method in which the second argument is a function, then you are using Higher Order Functions. You probably have used this before in your code, if you pass a callback function that is essentially a higher order function. Higher order functions can return a function. This is when you use composability. This is when you have code that is pretty repetitive and allows you to condense the code and avoid repeating code. So what makes them important? Javascript allows you to pass a function as an argument and this is unique as many other programming languages do not let you do this.


Array Methods

find()
This method will give you the value of the first array element that fits the function you provide(so it will give a true value) 

        var grades = [90, 20, 80, 75]

        function notPassingGrade(grade)
        {
            return grade < 70
        }
        
        grades.find(notPassingGrade);
        //20

The findIndex() method does essentially the same thing as find but will give you the array number as your result. 

        var grades = [60, 20, 80, 75]

        function notPassingGrade(grade)
        {
	        return grade > 70
        }
        
        //2

80 is the value that will return true in this situation and since its in the second location in the array it will therefore return 2


forEach() array method is used when you want to display each array element with their corresponding keys.


filter() will return all the values in the array that you define

sort() this method will, by default sort the values in an array as strings in alphabetical and ascending order. This is useful when sorting strings but if you are sorting numbers it will give you an incorrect answer. if you have the number 460 and 22, 22 will be sorted first because it starts with a 2 while the other number starts with a 4. 

    var colors = [“blue”, “yellow”, “orange”, “red”, “purple”];

    //blue, orange, purple, red, yellow

push() is used when you want to add a new element to an array but it will be added to the end of it. 

    var colors = [“blue”, “yellow”, “orange”, “red”, “purple”];

    colors.push(“pink”);

    //blue yellow orange red purple pink

if you want to add elements to the front of an array you will use the unshift() array method 

    var colors = [“blue”, “yellow”, “orange”, “red”, “purple”];

    colors.unshift(“pink”);

    //pink blue yellow orange red purple

shift() is used when you want to remove the first element in an array 

    var colors = [“blue”, “yellow”, “orange”, “red”, “purple”];

    colors.shift;
    
    //yellow orange red purple
